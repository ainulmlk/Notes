Étape 01
Core Network & Inter-VLAN Routing

Build the internal network backbone with a Multi-Layer Switch (MLS), access switches, and spanning tree configuration.

Key Concepts
VLAN (Virtual LAN)
Logically separates a physical network into isolated broadcast domains. VLAN10 = Engineering, VLAN20 = HR, VLAN30 = Servers. Devices in different VLANs cannot communicate without routing.
Multi-Layer Switch (MLS)
A switch that can also perform Layer 3 routing between VLANs using SVI (Switch Virtual Interfaces). Requires ip routing to be enabled. Acts as the default gateway for each VLAN.
Trunk vs Access Port
Trunk: carries multiple VLANs (dot1q tagging) — used between switches. Access: carries only one VLAN — used for end devices. Trunks should only allow necessary VLANs.
Spanning Tree Protocol (STP)
Prevents layer 2 loops by blocking redundant paths. The root bridge is elected based on priority (lower = wins). Force MLS as root with spanning-tree vlan X root primary.

Important Notes
⚠
Trunks must only allow necessary VLANs — leaving all VLANs open on a trunk is a security issue and will be penalized in grading.
ℹ
The no switchport command converts a switch port from Layer 2 to Layer 3 (routed mode), allowing an IP address to be assigned directly — used on Fa0/24 to connect MLS to ASA.
✓

Étape 02
ASA Integration + DNS/DHCP Services

Connect the ASA 5505 firewall between the core network and the FAI router. Deploy centralized DNS and DHCP services.

Key Concepts
ASA 5505 Architecture
The ASA 5505 uses VLANs internally — physical ports are assigned to VLANs, and each VLAN gets a logical interface (nameif). No IPs on physical ports directly.
Security Levels
inside = 100 (most trusted), dmz = 50, outside = 0 (least trusted). Traffic flows freely from high→low. Low→high requires explicit ACLs.
DHCP Relay (ip helper-address)
DHCP requests are broadcast — they can't cross routers. The ip helper-address on MLS SVIs converts broadcasts to unicast and forwards them to the DHCP server at 10.0.0.20.
DNS A Records
Type A record maps a hostname to an IPv4 address. We configured: www.entreprise.com → 192.168.1.10 and www.test.com → 8.8.0.10.
ASA Basic Configuration
ASA1 — Interface Setup
! Assign physical ports to VLANs interface Ethernet0/1 switchport access vlan 1 ! → MLS (inside) ! Configure logical VLAN interfaces interface Vlan1 nameif inside security-level 100 ip address 192.168.10.2 255.255.255.252
⚠
ASA 5505 license limit: only 2 interfaces can have nameif without restriction. The 3rd interface (DMZ) requires no forward interface Vlan1 BEFORE adding nameif — otherwise you get a license error.

Étape 03
DMZ + NAT + FAI Router

Add the DMZ web server, configure static and dynamic NAT, set firewall ACLs, and connect the FAI router.

Key Concepts
DMZ (Demilitarized Zone)
A network segment between inside and outside. Servers here are accessible from Internet but isolated from internal network. Uses no forward interface Vlan1 to prevent DMZ→inside traffic.
Static NAT
One-to-one permanent mapping: 1.1.1.253 ↔ 192.168.1.10. External clients use the public IP, ASA translates to the private DMZ IP. Bidirectional — works both ways automatically.
Dynamic NAT (PAT/Overload)
Many-to-one: all internal clients share one public IP. Uses port numbers to distinguish sessions. Configured with nat (inside,outside) dynamic interface.
Stateful Inspection
ASA tracks connection state. Traffic initiated from inside is allowed back automatically. Outside-initiated traffic to inside is blocked by default — key firewall behavior.

Common Problems & Fixes
✕
Double NAT problem: If static NAT is configured on BOTH ASA and Router_FAI, packets get translated twice — the destination IP changes to 192.168.1.10 at Router_FAI, then ASA doesn't recognize it. Fix: static NAT only on ASA.
✕
Subnet conflict: 1.1.1.253 is in the same /24 as Router_FAI's Fa0/0 (1.1.1.0/24), so Router_FAI thinks it's directly connected and doesn't forward. Fix: ip route 1.1.1.253 255.255.255.255 192.168.11.253
⚠
Missing return routes: ASA needs route inside commands to reach 172.16.x.x and 10.0.0.x. Otherwise it can't reply to internal clients.
Étape 04
Public Network + EIGRP Routing

Build the 8.8.0.0/16 external network and configure EIGRP dynamic routing between Router_FAI and Router_Externe.

Key Concepts
EIGRP
Enhanced Interior Gateway Routing Protocol — Cisco proprietary. Advertises directly connected networks to neighbors. Process number must match (both using 100). Uses no auto-summary for subnet accuracy.
Loopback Interface
A virtual interface that is always up. Used to make 1.1.1.0/24 "exist" as a connected network on Router_FAI so EIGRP can advertise it to Router_Externe — even though it's not physically wired.
no auto-summary
Prevents EIGRP from summarizing routes to classful boundaries (e.g., 1.1.2.128/30 → 1.0.0.0/8). Always use this in modern networks to advertise exact subnets.
External DNS
DNS server at 8.8.0.2 in the external network. Resolves www.entreprise.com → 1.1.1.253 so PC_Externe can access the DMZ web server by domain name.
