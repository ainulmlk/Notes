Étape 01
Cœur de réseau & Routage inter-VLAN
Barème : /5 points

Construire le réseau interne avec un Multi-Layer Switch (MLS), des switches d'accès, et configurer le Spanning Tree sans boucle.

Concepts Clés
VLAN (Virtual LAN)
Sépare logiquement un réseau physique en domaines de broadcast isolés. VLAN10 = clients groupe 1, VLAN20 = clients groupe 2, VLAN30 = serveurs. Les devices de VLANs différents ne communiquent pas sans routage.
Multi-Layer Switch (MLS)
Switch capable de faire du routage Layer 3 entre VLANs via des SVI. Nécessite ip routing pour activer le routage. Sert de passerelle par défaut pour chaque VLAN.
Port Trunk vs Access
Trunk : transporte plusieurs VLANs (802.1q) — entre switches. Access : un seul VLAN — pour les équipements finaux. Les trunks ne doivent laisser passer que les VLANs nécessaires (pénalité si non respecté).
STP (Spanning Tree Protocol)
Prévient les boucles Layer 2 en bloquant les chemins redondants. Le Root Bridge est élu par priorité (plus basse = gagne). On force le MLS comme racine pour VLAN10 et VLAN20.
SVI (Switch Virtual Interface)
Interface Layer 3 virtuelle créée avec interface vlan X sur le MLS. Sert de passerelle par défaut pour tous les hôtes du VLAN. Permet le routage inter-VLAN sans routeur externe.
no switchport
Convertit un port switch (Layer 2) en port routé (Layer 3). Utilisé sur Fa0/24 du MLS pour connecter le MLS à l'ASA avec une adresse IP directe — comme un port de routeur.

Étape 02
Intégration ASA + Services DNS/DHCP
Barème : /5 points (1+2+2)

Ajouter le pare-feu ASA 5505 entre le cœur réseau et le routeur FAI, mettre en place les services DNS et DHCP centralisés.

Concepts Clés
Architecture ASA 5505
Le 5505 utilise un switch interne : les ports physiques sont assignés à des VLANs (switchport access vlan X), et c'est l'interface VLAN logique qui reçoit l'IP et le nameif. On ne configure PAS d'IP sur les ports physiques.
Niveaux de Sécurité ASA
inside = 100 (réseau de confiance), dmz = 50 (semi-confiance), outside = 0 (non fiable). Le trafic circule librement de haut→bas. Bas→haut nécessite une ACL explicite.
ip helper-address (Relay DHCP)
DHCP utilise le broadcast — ne traverse pas les routeurs. ip helper-address 10.0.0.20 sur les SVIs du MLS convertit le broadcast en unicast vers le serveur DHCP. Un seul serveur pour tous les VLANs.
Enregistrements DNS de type A
Un enregistrement A mappe un nom d'hôte vers une IPv4. Configurés : www.entreprise.com → 192.168.1.10 (futur serveur DMZ) et www.test.com → 8.8.0.10 (serveur externe).
Configuration ASA — Interface Inside
ASA1 — Étape 2
! Assigner le port physique vers MLS au VLAN1 (inside) interface Ethernet0/1 switchport access vlan 1 ! Configurer l'interface logique inside interface Vlan1 nameif inside security-level 100 ip address 192.168.10.2 255.255.255.252 no shutdown
⚠
Limitation licence ASA 5505 : seulement 2 interfaces peuvent avoir nameif sans restriction. La 3ème (DMZ) nécessite no forward interface Vlan1 AVANT nameif — sinon erreur de licence.

Étape 03
DMZ + NAT + Routeur FAI
Barème : /10 points (4 NAT + 4 pare-feu + 2 DMZ/licence)

Connecter l'ASA au routeur FAI, créer la DMZ avec le serveur web, configurer le NAT statique/dynamique et les règles pare-feu.

Concepts Clés
DMZ (Zone Démilitarisée)
Zone réseau entre inside et outside. Les serveurs DMZ sont accessibles depuis Internet mais isolés du réseau interne. no forward interface Vlan1 empêche la DMZ d'initier des connexions vers l'inside.
NAT Statique
Mappage permanent 1-pour-1 : 1.1.1.253 ↔ 192.168.1.10. Bidirectionnel automatiquement. Les clients externes utilisent l'IP publique, l'ASA traduit vers l'IP privée du serveur DMZ.
NAT Dynamique (PAT/Overload)
Plusieurs-vers-un : tous les clients internes partagent une seule IP publique, distingués par les numéros de port. Configuré avec nat (inside,outside) dynamic interface.
Inspection Stateful
L'ASA maintient une table de connexions. Trafic initié depuis l'inside autorisé en retour automatiquement. Trafic initié depuis l'outside vers l'inside bloqué par défaut.
ACL sur ASA 8.3+
Depuis ASA 8.3, l'ASA dé-NAT le paquet AVANT de vérifier l'ACL. Les ACLs doivent référencer l'adresse IP réelle (192.168.1.10), pas l'adresse NATée (1.1.1.253).
Double NAT — Problème
Si NAT statique est sur ASA ET Router_FAI : Router_FAI traduit 1.1.1.253→192.168.1.10, l'ASA reçoit une IP privée qu'il ne reconnaît pas → paquet abandonné. Solution : NAT statique uniquement sur l'ASA.

Problèmes Rencontrés et Solutions
✕
Double NAT : NAT statique sur ASA ET Router_FAI → double translation → paquet abandonné. Solution : NAT statique uniquement sur l'ASA.
✕
Conflit de sous-réseau : 1.1.1.253 dans le même /24 que Fa0/0 du Router_FAI (1.1.1.0/24) → Router_FAI boucle. Solution : ip route 1.1.1.253 255.255.255.255 192.168.11.253
✕
Erreur licence DMZ : nameif dmz avant no forward → erreur de licence. Solution : ordre obligatoire : no forward interface Vlan1 puis nameif dmz.
⚠
Routes manquantes sur l'ASA : sans route inside 172.16.x.x, l'ASA ne sait pas retourner les réponses aux clients internes.
⚠
Vlan2 up/down : câble sur le mauvais port ou Router_FAI pas encore configuré → lien ne monte pas. Vérifier show interface ip brief sur l'ASA.
ℹ
inspect icmp obligatoire : sans inspect icmp dans la policy-map, les réponses aux pings sont bloquées car l'ASA ne suit pas l'état des sessions ICMP.

Étape 04
Réseau Public 8.8.0.0/16 + EIGRP
Barème : /5 points

Construire le réseau externe 8.8.0.0/16, configurer EIGRP entre Router_FAI et Router_Externe, valider les tests finaux.

Concepts Clés
EIGRP
Enhanced Interior Gateway Routing Protocol — propriétaire Cisco. N'annonce que les réseaux directement connectés. Le numéro de processus doit correspondre entre voisins (ici : 100). Plus simple qu'OSPF pour les petits réseaux Cisco.
Interface Loopback
Interface virtuelle toujours active. Utilisée pour rendre 1.1.1.0/24 "connecté" sur Router_FAI afin qu'EIGRP puisse l'annoncer à Router_Externe. Sans loopback, EIGRP ne peut pas annoncer ce réseau.
no auto-summary
Empêche EIGRP de résumer les routes vers les frontières classful (ex: 1.1.2.128/30 → 1.0.0.0/8). Toujours utiliser pour annoncer les sous-réseaux exacts.
DNS Externe
Serveur à 8.8.0.2. Résout www.entreprise.com → 1.1.1.253 pour que PC_Externe accède au serveur web de l'entreprise par nom de domaine.

Référence
Toutes les Commandes

Référence complète de chaque commande utilisée dans le projet SAÉ21.

Commandes MLS et Switches
ip routing
Active le routage Layer 3 sur le MLS. Sans cette commande, le MLS ne peut pas router entre les VLANs.
no switchport
Convertit un port switch en port routé L3. Utilisé sur Fa0/24 pour connecter le MLS à l'ASA avec une IP.
ip helper-address X
Relay DHCP sur les SVIs. Convertit le broadcast DHCP en unicast vers le serveur 10.0.0.20.
spanning-tree vlan X root primary
Force le MLS comme Root Bridge pour le VLAN X en abaissant automatiquement la priorité STP.
switchport mode trunk
Configure le port en mode trunk — transporte plusieurs VLANs avec le tagging 802.1q.
switchport trunk allowed vlan X
Limite les VLANs autorisés sur le trunk. Essentiel pour la sécurité — ne laisser que le nécessaire.
switchport mode access
Configure le port en mode access — un seul VLAN, pour les équipements finaux (PCs, serveurs).
spanning-tree portfast
Supprime le délai STP sur les ports connectés directement à des hôtes (jamais à d'autres switches).
Commandes ASA
nameif [inside|outside|dmz]
Donne un nom fonctionnel à l'interface VLAN. Obligatoire pour toutes les règles NAT/ACL.
security-level [0-100]
Niveau de confiance de l'interface. 100=inside, 50=dmz, 0=outside. Contrôle les flux par défaut.
no forward interface VlanX
Interdit à cette interface d'initier des connexions vers VlanX. DOIT précéder nameif sur ASA 5505.
object network NAME / host X
Crée un objet réseau représentant un hôte ou sous-réseau. Utilisé dans les règles NAT.
nat (dmz,outside) static X
NAT statique bidirectionnel : IP privée DMZ ↔ IP publique X. Pour le ServeurWeb.
nat (inside,outside) dynamic interface
PAT (overload) : traduit les IPs internes vers l'IP de l'interface outside de l'ASA.
access-list NAME extended permit tcp any host X eq www
Autorise HTTP depuis n'importe quelle source vers l'hôte X. Référence l'IP réelle (pas NATée).
access-group NAME in interface X
Applique une ACL en entrée sur une interface. Le trafic entrant est filtré selon les règles.
route outside 0.0.0.0 0.0.0.0 X
Route par défaut — tout trafic inconnu est envoyé vers Router_FAI (192.168.11.254).
route inside 172.16.x.x X.X.X.X nexthop
Route statique vers les réseaux internes via le MLS. Nécessaire pour les réponses NAT.
class-map inspection_default
Définit la classe de trafic à inspecter. Associé à match default-inspection-traffic.
inspect icmp / inspect http
Active l'inspection stateful pour ICMP et HTTP. Permet les réponses aux connexions initiées depuis l'inside.
service-policy global_policy global
Applique la policy-map sur toutes les interfaces globalement.
show interface ip brief
Affiche le statut de toutes les interfaces ASA. up/up = OK, up/down = problème physique.
show nat / show access-list
Affiche les règles NAT (translate_hits) et ACLs (hitcnt) — vérifier que le trafic passe.
show switch vlan
Affiche les VLANs internes de l'ASA 5505 et les ports physiques associés à chaque VLAN.
Commandes Router IOS (Router_FAI / Router_Externe)
ip nat inside / ip nat outside
Marque les interfaces comme intérieure/extérieure pour le NAT. Obligatoire pour que NAT fonctionne.
ip nat inside source static A B
NAT statique : mappe l'adresse privée A vers l'adresse publique B. Permanent et bidirectionnel.
ip nat inside source list X interface Y overload
PAT : traduit les adresses de l'ACL X vers l'IP de l'interface Y. Overload = plusieurs IPs partagent une seule IP publique.
ip route X X.X.X.X nexthop
Route statique. X/masque = destination, nexthop = prochain saut.
router eigrp 100
Active le processus EIGRP 100. Le numéro DOIT correspondre entre tous les voisins.
network X wildcard
Annonce le réseau X dans EIGRP. Le masque est en wildcard (inverse du masque de sous-réseau).
no auto-summary
Désactive le résumé automatique classful. EIGRP annonce les sous-réseaux exacts configurés.
interface Loopback1 / ip address X
Interface virtuelle toujours active. Utilisée pour rendre 1.1.1.0/24 "connecté" sur Router_FAI.
show ip eigrp neighbors
Affiche les voisins EIGRP actifs. Chaque routeur doit voir l'autre dans cette liste.
show ip nat translations
Affiche la table NAT active — vérifier que les translations fonctionnent correctement.
show ip route
Table de routage. C=connecté, S=statique, D=EIGRP. Vérifier que toutes les routes attendues sont présentes.
